# Dio — next-session seed

Read this once, then proceed.

## Project in 30 seconds

Multiplayer arcade racer in Unity 6 + URP, networking via **Mirror** (vendored at `Assets/Mirror/`, gitignored). Tracks are geodesic strips around a small planet. Car physics: server-authoritative WheelCollider + custom spherical gravity. UI built programmatically by `Tools → Dio → Build → All`.

The full design + audit lives in [plan.md](./plan.md). Recent run-through and the pain-point list is below.

## What's done (don't redo)

The full audit is in [plan.md §11](./plan.md). Quick summary of fixed items:

- Wheels touch ground; visuals spin/steer locally on every peer (driven by `transform` velocity + `DioCar.steerAngle` SyncVar)
- Remote rb config is strict: `kinematic + None + Discrete`
- LAN discovery enumerates every up IPv4 adapter and broadcasts to each subnet's specific broadcast (works around WARP / VPN)
- Direct-IP fallback in browser panel; IP cached in `LocalPrefs.LastDirectIp`; Enter triggers Connect
- Powerup row group constraint (`PowerupBox.groupId` + `PowerupHolder.consumedGroups` SyncList) — one consume per pass
- Win condition: server polls car distance to `level.Finish`; sends `RaceWonMessage`; locks all inputs; `ServerCleanupRace` destroys per-race objects after 5.5s; clients restore lobby
- Win UI: `winner_banner.svg` + `UIConfetti` (UI-Image-based, no extra camera)
- Minimap is a 256×256 RT from `MinimapRenderer`; ortho 130; UI markers projected via `WorldToViewportPoint` with hemisphere cull; panel 330 px
- Procedural shaders: `Dio/CarBodyCamo`, `Dio/CarWheel`, `Dio/EarthPlanet`, `Dio/MysteryBox`, `Dio/SunsetSky`. All use `_DioTime` (NetworkTime) + `_PlayerUp` globals so visuals are consistent across peers
- Vehicle parametrised by `VehicleProfile` ScriptableObject (mesh + dimensions + suspension)
- Track strip mesh from `TrackBuilder.Build(level)`; LineRenderer removed; planet + track on Layer 8 (`MinimapLayer`)

## Open / unverified pain points

These are the ones the user wants you to investigate with the MCPs:

1. **Joined-player physics still felt frozen on their screen** in earlier tests. Latest fix is the strict kinematic config in `DioCar.OnStartClient`. Verify with the user that it works. If it doesn't:
   - Use `mirror-docs` to query: "For a kinematic client receiving server-authoritative NetworkTransform updates, what Rigidbody settings does Mirror recommend? Is NetworkRigidbody preferred?"
   - Use `unity-mcp` to inspect a running joined client's `DioCar` GameObject during a race: confirm `Rigidbody.isKinematic == true`, `NetworkIdentity.connectionToClient` matches, `transform.position` is changing.
2. **LAN auto-discovery over Cloudflare WARP** — multi-adapter broadcast is in `DioNetworkDiscovery.MultiAdapterBroadcast`. Verify `[Dio Discovery] server got ping from …` shows in the host's console when a client clicks *Join Game*. If not, the broadcast isn't reaching the host. Use `unity-mcp` to grep the console.
3. **Lap detection / proper race position** — currently the position panel + win condition use **closest-distance to finish** as a stand-in. Real implementation is M2 (arc-length-along-track tracker). Don't ship without telling the user.
4. **Predicted rigidbody for the local player** (input-lag fix) — Mirror has `PredictedRigidbody` example at `Assets/Mirror/Examples/BilliardsPredicted/`. M3 milestone in plan.md.

## How to use the MCPs efficiently

- **`mirror-docs`** — fact-check Mirror's APIs and recommended patterns *before* writing/changing networked code. Especially for `NetworkTransform`, `NetworkRigidbody`, `NetworkDiscovery`, `[Command]` authority rules.
- **`unity-mcp`** — read scene state, inspector values, console logs while Unity is running. Don't use it to write/edit assets that the user hasn't asked you to touch — the editor menus are the canonical write path.

## Editing rules

- **Don't write `.meta` files into the repo.** They're gitignored on purpose.
- **Don't reintroduce `Library/PackageCache/` paths in shaders.** Use `UnityCG.cginc` (legacy) for skybox + procedural shaders so they work on both URP and Built-in.
- **Don't break the editor-menu rebuild flow.** Anything generated should be re-runnable by `Tools → Dio → Build → All` without hand-editing prefabs.
- **Always check `git status --short` before `git add -A`.** Several Unity auto-generated assets at `Assets/`-root and `Assets/Resources/` will sneak in otherwise.
